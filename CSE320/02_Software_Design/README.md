# Module 2: Software Design Principles

## Overview
This module focuses on fundamental software design principles, with emphasis on achieving high cohesion and low coupling - the cornerstone of good software design.

## Topics Covered

### 1. Design Principles
- Abstraction and refinement
- Modularity
- Information hiding
- Design trade-offs
- **File**: [01_Design_Principles.md](01_Design_Principles.md)

### 2. Cohesion and Coupling ⭐ **VERY IMPORTANT**
- Types of cohesion (7 types)
- Types of coupling (6 types)
- Measuring module independence
- Design rules
- **File**: [02_Cohesion_and_Coupling.md](02_Cohesion_and_Coupling.md)

### 3. DFD and Structured Analysis
- Data Flow Diagrams
- Level 0, 1, 2 DFDs
- Context diagrams
- **File**: [03_DFD_and_Structured_Analysis.md](03_DFD_and_Structured_Analysis.md)

### 4. Structure Charts
- Components and notation
- Transform analysis
- Transaction analysis
- **File**: [04_Structure_Charts.md](04_Structure_Charts.md)

## Course Outcome: CO2
After completing this module, you will be able to:
- Apply software design principles to evaluate system architectures
- Analyze and improve module independence
- Create DFDs and structure charts for system design
