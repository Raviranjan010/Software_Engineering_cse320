# Software Engineering MCQs - Module 1: SDLC and Requirements Engineering

## Part 1: Introduction to Software Engineering

**Q1. Software is defined as ___________**  
a) set of programs, documentation & configuration of data  
b) set of programs  
c) documentation and configuration of data  
d) None of the mentioned  
**Answer: a) set of programs, documentation & configuration of data**  
*Explanation: Software includes programs, documentation, and data configuration.*

**Q2. What is Software Engineering?**  
a) Designing a software  
b) Testing a software  
c) Application of engineering principles to the design a software  
d) None of the above  
**Answer: c) Application of engineering principles to the design a software**  
*Explanation: Software Engineering applies systematic, disciplined, quantifiable approaches to software development.*

**Q3. Who is the father of Software Engineering?**  
a) Margaret Hamilton  
b) Watts S. Humphrey  
c) Alan Turing  
d) Boris Beizer  
**Answer: a) Margaret Hamilton**  
*Explanation: Margaret Hamilton coined the term "Software Engineering" at the 1968 NATO Conference.*

**Q4. What are the features of Software Code?**  
a) Simplicity  
b) Accessibility  
c) Modularity  
d) All of the above  
**Answer: d) All of the above**  
*Explanation: Good software code should be simple, accessible, modular, maintainable, and efficient.*

**Q5. Software Engineering emerged as a discipline due to:**  
a) Hardware crisis  
b) Software crisis  
c) Network crisis  
d) Database crisis  
**Answer: b) Software crisis**  
*Explanation: The software crisis of 1968 led to the formal establishment of Software Engineering.*

**Q6. Which of the following is NOT a software characteristic?**  
a) Software is engineered, not manufactured  
b) Software doesn't wear out  
c) Software is simple  
d) Software is complex  
**Answer: c) Software is simple**  
*Explanation: Software is inherently complex, not simple.*

**Q7. The term "Software Engineering" was coined at which conference?**  
a) IEEE Conference 1965  
b) NATO Conference 1968  
c) ACM Conference 1970  
d) UN Conference 1972  
**Answer: b) NATO Conference 1968**  

**Q8. Which era of software development featured cloud computing and AI?**  
a) Batch Era  
b) PC Era  
c) Internet Era  
d) Cloud/AI Era  
**Answer: d) Cloud/AI Era**  
*Explanation: 2010s+ is characterized by cloud computing, AI, microservices, and DevOps.*

**Q9. Software failure curve differs from hardware because:**  
a) Software has infant mortality  
b) Software has wear-out phase  
c) Software doesn't wear out physically  
d) Software follows bathtub curve  
**Answer: c) Software doesn't wear out physically**  
*Explanation: Software doesn't physically degrade but deteriorates due to changes.*

**Q10. Which of the following was NOT a problem during the software crisis?**  
a) Projects over budget  
b) Projects behind schedule  
c) Software was too reliable  
d) Difficult to maintain software  
**Answer: c) Software was too reliable**  
*Explanation: Software was unreliable and buggy during the crisis.*

**Q11. The cost pattern of software is characterized by:**  
a) High per-unit cost  
b) High development, low reproduction cost  
c) Equal development and manufacturing cost  
d) No development cost  
**Answer: b) High development, low reproduction cost**  

**Q12. Unlike hardware, software cannot be:**  
a) Modified  
b) Assembled from spare parts  
c) Tested  
d) Designed  
**Answer: b) Assembled from spare parts**  

**Q13. Which is NOT a benefit of studying Software Engineering?**  
a) Quality software production  
b) Predictable cost and time  
c) Guaranteed bug-free software  
d) Better teamwork  
**Answer: c) Guaranteed bug-free software**  
*Explanation: SE reduces bugs but cannot guarantee bug-free software.*

**Q14. Software deterioration is caused by:**  
a) Physical wear and tear  
b) Environmental factors  
c) Changes and updates  
d) Time passage  
**Answer: c) Changes and updates**  

**Q15. The batch era of software development was characterized by:**  
a) Multiple programs in memory  
b) Personal computers  
c) Single-user, batch processing  
d) Web applications  
**Answer: c) Single-user, batch processing**

## Part 2: SDLC Models

**Q16. What does SDLC stand for?**  
a) System Design Life Cycle  
b) Software Design Life Cycle  
c) Software Development Life Cycle  
d) System Development Life cycle  
**Answer: c) Software Development Life Cycle**  

**Q17. Which SDLC model follows a linear sequential approach?**  
a) Agile  
b) Spiral  
c) Waterfall  
d) Prototyping  
**Answer: c) Waterfall**  

**Q18. Who proposed the spiral model?**  
a) Barry Boehm  
b) Pressman  
c) Royce  
d) IBM  
**Answer: a) Barry Boehm**  
*Explanation: Barry Boehm proposed the Spiral Model in 1988.*

**Q19. Which model is best suited for high-risk projects?**  
a) Waterfall  
b) Prototyping  
c) Spiral  
d) V-Model  
**Answer: c) Spiral**  
*Explanation: Spiral model explicitly includes risk analysis in each cycle.*

**Q20. In which model does testing happen parallel to development?**  
a) Waterfall  
b) Agile  
c) V-Model  
d) Prototyping  
**Answer: c) V-Model**  
*Explanation: V-Model has corresponding testing phase for each development phase.*

**Q21. Sprint is a concept in which methodology?**  
a) Waterfall  
b) Spiral  
c) Scrum  
d) V-Model  
**Answer: c) Scrum**  
*Explanation: Sprint is a time-boxed iteration (2-4 weeks) in Scrum framework.*

**Q22. Which is NOT an Agile value?**  
a) Working software over documentation  
b) Contract negotiation over collaboration  
c) Responding to change over following plan  
d) Individuals over processes  
**Answer: b) Contract negotiation over collaboration**  
*Explanation: Agile values customer collaboration OVER contract negotiation.*

**Q23. DevOps combines which two functions?**  
a) Design and Operations  
b) Development and Operations  
c) Testing and Deployment  
d) Planning and Coding  
**Answer: b) Development and Operations**  

**Q24. Which model is NOT suitable for accommodating changes?**  
a) RAD Model  
b) Waterfall Model  
c) Build & Fix Model  
d) Prototyping Model  
**Answer: b) Waterfall Model**  
*Explanation: Waterfall has low flexibility and difficult to go back to previous phases.*

**Q25. The prototype model is best when:**  
a) Requirements are clear  
b) Requirements are unclear  
c) Project is small  
d) Technology is stable  
**Answer: b) Requirements are unclear**  

**Q26. How many quadrants are in the Spiral Model?**  
a) 2  
b) 3  
c) 4  
d) 5  
**Answer: c) 4**  
*Explanation: Planning, Risk Analysis, Engineering, Evaluation.*

**Q27. Which SDLC phase produces the SRS document?**  
a) Planning  
b) Requirement Analysis  
c) Design  
d) Testing  
**Answer: b) Requirement Analysis**  

**Q28. In V-Model, acceptance testing corresponds to:**  
a) System Design  
b) Requirements Analysis  
c) Module Design  
d) Coding  
**Answer: b) Requirements Analysis**  

**Q29. Which model delivers working software at the end of each iteration?**  
a) Waterfall  
b) V-Model  
c) Agile  
d) Build & Fix  
**Answer: c) Agile**  

**Q30. The Daily Scrum meeting duration is:**  
a) 30 minutes  
b) 1 hour  
c) 15 minutes  
d) 2 hours  
**Answer: c) 15 minutes**  

**Q31. Which is NOT a Scrum role?**  
a) Product Owner  
b) Scrum Master  
c) Project Manager  
d) Development Team  
**Answer: c) Project Manager**  

**Q32. The main focus of the Spiral model is:**  
a) Speed  
b) Risk Analysis  
c) Documentation  
d) Testing  
**Answer: b) Risk Analysis**  

**Q33. Agile Software Development is based on:**  
a) Iterative Development  
b) Incremental Development  
c) Both Incremental and Iterative Development  
d) Linear Development  
**Answer: c) Both Incremental and Iterative Development**  

**Q34. Which DevOps stage comes after "Code"?**  
a) Plan  
b) Build  
c) Test  
d) Deploy  
**Answer: b) Build**  

**Q35. The Waterfall model is also known as:**  
a) Iterative Model  
b) Classic Life Cycle  
c) Agile Model  
d) Spiral Model  
**Answer: b) Classic Life Cycle**  

**Q36. In which model is customer involvement the highest?**  
a) Waterfall  
b) V-Model  
c) Agile/Prototyping  
d) Spiral  
**Answer: c) Agile/Prototyping**  

**Q37. Throwaway prototype is:**  
a) Refined into final product  
b) Discarded after requirements clarification  
c) Used for production  
d) Final system  
**Answer: b) Discarded after requirements clarification**  

**Q38. Which model is best for safety-critical systems?**  
a) Agile  
b) Waterfall  
c) V-Model  
d) Prototyping  
**Answer: c) V-Model**  
*Explanation: V-Model has parallel testing ensuring thorough verification.*

**Q39. The number of members in a Scrum development team is typically:**  
a) 1-3  
b) 5-9  
c) 10-15  
d) 15-20  
**Answer: b) 5-9**  

**Q40. CI/CD in DevOps stands for:**  
a) Continuous Integration/Continuous Deployment  
b) Central Integration/Central Deployment  
c) Continuous Improvement/Continuous Development  
d) Code Integration/Code Deployment  
**Answer: a) Continuous Integration/Continuous Deployment**

## Part 3: Requirements Engineering

**Q41. What is a Functional Requirement?**  
a) specifies the tasks the program must complete  
b) specifies the tasks the program should not complete  
c) specifies the tasks the program must not work  
d) All of the mentioned  
**Answer: a) specifies the tasks the program must complete**  

**Q42. "The system shall respond within 2 seconds" is an example of:**  
a) Functional requirement  
b) Non-functional requirement  
c) Design constraint  
d) Business rule  
**Answer: b) Non-functional requirement**  

**Q43. In FURPS model, 'U' stands for:**  
a) Utility  
b) Usability  
c) Uniformity  
d) Uniqueness  
**Answer: b) Usability**  

**Q44. Which requirement gathering technique is best for large user bases?**  
a) Interviews  
b) Observation  
c) Questionnaires  
d) Prototyping  
**Answer: c) Questionnaires**  

**Q45. Which word indicates a mandatory requirement in SRS?**  
a) Should  
b) May  
c) Shall  
d) Could  
**Answer: c) Shall**  

**Q46. JAD stands for:**  
a) Joint Application Development  
b) Joint Analysis Design  
c) Java Application Development  
d) Joint Agile Development  
**Answer: a) Joint Application Development**  

**Q47. Which is NOT a requirement validation technique?**  
a) Requirements Review  
b) Prototyping  
c) Coding  
d) Test-Case Generation  
**Answer: c) Coding**  

**Q48. RTM stands for:**  
a) Requirements Testing Method  
b) Requirements Traceability Matrix  
c) Real-Time Monitoring  
d) Requirement Tracking Mechanism  
**Answer: b) Requirements Traceability Matrix**  

**Q49. In MoSCoW method, 'M' stands for:**  
a) Must Have  
b) May Have  
c) Major  
d) Minimum  
**Answer: a) Must Have**  

**Q50. Which is a good requirement?**  
a) System should be fast  
b) System shall respond within 2 seconds  
c) System should be user-friendly  
d) System should handle large data  
**Answer: b) System shall respond within 2 seconds**  

**Q51. "System shall support 1000 concurrent users" is which type of requirement?**  
a) Functional  
b) Performance (Non-functional)  
c) Design  
d) Interface  
**Answer: b) Performance (Non-functional)**  

**Q52. Which requirement gathering technique involves watching users in their environment?**  
a) Interviews  
b) Observation  
c) Questionnaires  
d) Brainstorming  
**Answer: b) Observation**  

**Q53. The FURPS model category 'R' stands for:**  
a) Response  
b) Reliability  
c) Robustness  
d) Reusability  
**Answer: b) Reliability**  

**Q54. Which is NOT a characteristic of good requirements?**  
a) Unambiguous  
b) Verifiable  
c) Vague  
d) Complete  
**Answer: c) Vague**  

**Q55. Requirements validation ensures:**  
a) Code is bug-free  
b) Requirements are correct, complete, and consistent  
c) Testing is complete  
d) Deployment is successful  
**Answer: b) Requirements are correct, complete, and consistent**

## Part 4: SRS Document

**Q56. Which document contains the user system requirements?**  
a) SRD  
b) DDD  
c) SDD  
d) SRS  
**Answer: d) SRS**  
*Explanation: SRS (Software Requirements Specification) contains system requirements.*

**Q57. SRS specification is also known as:**  
a) white-box  
b) grey-box  
c) black-box  
d) none of the mentioned  
**Answer: c) black-box**  
*Explanation: SRS describes external behavior (black-box view) without implementation details.*

**Q58. IEEE standard for SRS is:**  
a) IEEE 808  
b) IEEE 830  
c) IEEE 1012  
d) IEEE 12207  
**Answer: b) IEEE 830**  

**Q59. Which section of SRS contains detailed functional requirements?**  
a) Section 1  
b) Section 2  
c) Section 3  
d) Section 4  
**Answer: c) Section 3**  

**Q60. SRS acts as a contract between:**  
a) Developer and Tester  
b) Customer and Developer  
c) Project Manager and Team  
d) Designer and Developer  
**Answer: b) Customer and Developer**  

**Q61. Which is NOT a characteristic of good SRS?**  
a) Correct  
b) Unambiguous  
c) Ambiguous  
d) Verifiable  
**Answer: c) Ambiguous**  

**Q62. In SRS, "should" indicates:**  
a) Mandatory requirement  
b) Desirable requirement  
c) Optional requirement  
d) Future requirement  
**Answer: b) Desirable requirement**  

**Q63. The "Overall Description" is which section of IEEE 830 SRS?**  
a) Section 1  
b) Section 2  
c) Section 3  
d) Appendix  
**Answer: b) Section 2**  

**Q64. Which is included in SRS Section 1 (Introduction)?**  
a) Functional Requirements  
b) Purpose and Scope  
c) Performance Requirements  
d) Design Constraints  
**Answer: b) Purpose and Scope**  

**Q65. RTM helps in:**  
a) Writing code  
b) Tracking requirements through development lifecycle  
c) Testing software  
d) Deploying software  
**Answer: b) Tracking requirements through development lifecycle**

## Part 5: Software Engineering Ethics and Professional Practice

**Q66. ______________ is not among the eight principles followed by the Software Code of Ethics and Professional Practice.**  
a) PRODUCT  
b) ENVIRONMENT  
c) PUBLIC  
d) PROFESSION  
**Answer: b) ENVIRONMENT**  
*Explanation: The 8 principles are: PUBLIC, CLIENT/EMPLOYER, PRODUCT, JUDGMENT, MANAGEMENT, PROFESSION, COLLEAGUES, SELF.*

**Q67. Engineers developing software should not:**  
a) be dependent on their colleagues  
b) maintain integrity and independence in their professional judgment  
c) not knowingly accept work that is outside your competence  
d) not use your technical skills to misuse other people's computers  
**Answer: a) be dependent on their colleagues**  
*Explanation: Engineers should maintain independence and integrity.*

**Q68. According to IBM research, what percentage of projects are abandoned before completion?**  
a) 21%  
b) 31%  
c) 41%  
d) 51%  
**Answer: b) 31%**  

**Q69. Why do bugs and failures occur in software?**  
a) Because of Developers  
b) Because of companies  
c) Because of both companies and Developers  
d) None of the mentioned  
**Answer: c) Because of both companies and Developers**
