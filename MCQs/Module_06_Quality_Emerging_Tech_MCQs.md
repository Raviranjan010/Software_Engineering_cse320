# Software Engineering MCQs - Module 6: Quality and Emerging Technologies

## Part 1: Quality Standards

**Q1. SQA stands for:**  
a) Software Quality Assurance  
b) System Quality Analysis  
c) Software Query Assessment  
d) System Quality Assurance  
**Answer: a) Software Quality Assurance**  

**Q2. SQM stands for:**  
a) Software Quality Management  
b) System Query Management  
c) Software Query Method  
d) System Quality Measurement  
**Answer: a) Software Quality Management**  

**Q3. Quality Management is known as:**  
a) SQI  
b) SQA  
c) SQM  
d) SQA and SQM  
**Answer: d) SQA and SQM**  

**Q4. ISO 9001 is a standard for:**  
a) Software coding  
b) Quality management systems  
c) Testing procedures  
d) Project management  
**Answer: b) Quality management systems**  

**Q5. SEI CMMI has how many maturity levels?**  
a) 3  
b) 4  
c) 5  
d) 6  
**Answer: c) 5**  

**Q6. CMMI Level 1 is:**  
a) Managed  
b) Initial  
c) Defined  
d) Optimizing  
**Answer: b) Initial**  

**Q7. CMMI Level 5 is:**  
a) Initial  
b) Repeatable  
c) Defined  
d) Optimizing  
**Answer: d) Optimizing**  

**Q8. The correct order of CMMI levels is:**  
a) Initial, Managed, Defined, Quantitatively Managed, Optimizing  
b) Initial, Repeatable, Defined, Managed, Optimizing  
c) Initial, Defined, Managed, Optimizing, Quantitatively Managed  
d) Initial, Managed, Optimizing, Defined, Quantitatively Managed  
**Answer: a) Initial, Managed, Defined, Quantitatively Managed, Optimizing**  

**Q9. At CMMI Level 2 (Managed):**  
a) Processes are unpredictable and chaotic  
b) Projects are planned and tracked  
c) Processes are standardized across organization  
d) Continuous process improvement  
**Answer: b) Projects are planned and tracked**  

**Q10. At CMMI Level 3 (Defined):**  
a) Processes are chaotic  
b) Processes are documented and standardized organization-wide  
c) Only projects are managed  
d) No improvement happening  
**Answer: b) Processes are documented and standardized organization-wide**  

**Q11. Six Sigma aims for:**  
a) 99% defect-free  
b) 99.9% defect-free  
c) 99.99966% defect-free (3.4 defects per million)  
d) 100% defect-free  
**Answer: c) 99.99966% defect-free (3.4 defects per million)**  

**Q12. Six Sigma uses which methodology?**  
a) Agile  
b) DMAIC (Define, Measure, Analyze, Improve, Control)  
c) Waterfall  
d) Scrum  
**Answer: b) DMAIC (Define, Measure, Analyze, Improve, Control)**  

**Q13. PSP stands for:**  
a) Personal Software Process  
b) Professional Software Practice  
c) Project Software Planning  
d) Program Software Process  
**Answer: a) Personal Software Process**  

**Q14. PSP helps developers to:**  
a) Manage projects  
b) Improve individual performance through measurement  
c) Test software  
d) Design systems  
**Answer: b) Improve individual performance through measurement**  

**Q15. Which is NOT a CMMI maturity level?**  
a) Initial  
b) Defined  
c) Optimizing  
d) Testing  
**Answer: d) Testing**

## Part 2: Software Maintenance

**Q16. Software maintenance occurs:**  
a) Before development  
b) During development  
c) After deployment  
d) Only during testing  
**Answer: c) After deployment**  

**Q17. How many types of software maintenance are there?**  
a) 2  
b) 3  
c) 4  
d) 5  
**Answer: c) 4**  

**Q18. Corrective Maintenance is for:**  
a) Adding new features  
b) Fixing bugs and errors  
c) Improving performance  
d) Preventing future problems  
**Answer: b) Fixing bugs and errors**  

**Q19. Adaptive Maintenance is for:**  
a) Fixing bugs  
b) Adapting to new environment (OS, hardware, regulations)  
c) Adding features  
d) Preventing issues  
**Answer: b) Adapting to new environment (OS, hardware, regulations)**  

**Q20. Perfective Maintenance is for:**  
a) Fixing defects  
b) Enhancing functionality and performance based on user requests  
c) Adapting to environment  
d) Preventive measures  
**Answer: b) Enhancing functionality and performance based on user requests**  

**Q21. Preventive Maintenance is for:**  
a) Fixing current bugs  
b) Making future maintenance easier  
c) Adding new features  
d) Adapting to new OS  
**Answer: b) Making future maintenance easier**  

**Q22. Which maintenance type consumes the MOST effort?**  
a) Corrective  
b) Adaptive  
c) Perfective  
d) Preventive  
**Answer: c) Perfective**  

**Q23. Which maintenance type consumes the LEAST effort?**  
a) Corrective  
b) Perfective  
c) Preventive  
d) Adaptive  
**Answer: c) Preventive**  

**Q24. Reverse Engineering is:**  
a) Creating new software from scratch  
b) Analyzing existing software to understand its design  
c) Testing software  
d) Deploying software  
**Answer: b) Analyzing existing software to understand its design**  

**Q25. Re-engineering is:**  
a) Analyzing software only  
b) Reconstructing software to improve it  
c) Deleting software  
d) Testing software  
**Answer: b) Reconstructing software to improve it**  

**Q26. The process of generating analysis and design documents from existing code is:**  
a) Forward engineering  
b) Reverse engineering  
c) Testing  
d) Deployment  
**Answer: b) Reverse engineering**  

**Q27. Software re-engineering includes:**  
a) Only reverse engineering  
b) Reverse engineering + restructuring + forward engineering  
c) Only forward engineering  
d) Only testing  
**Answer: b) Reverse engineering + restructuring + forward engineering**  

**Q28. Legacy systems are:**  
a) New systems  
b) Old systems still in use  
c) Testing systems  
d) Prototype systems  
**Answer: b) Old systems still in use**  

**Q29. Regression testing is important during maintenance to:**  
a) Test new features only  
b) Ensure changes don't break existing functionality  
c) Test performance  
d) Test security  
**Answer: b) Ensure changes don't break existing functionality**  

**Q30. Maintenance effort typically consumes what percentage of total software cost?**  
a) 10-20%  
b) 30-40%  
c) 60-75%  
d) 90-100%  
**Answer: c) 60-75%**

## Part 3: CASE Tools

**Q31. CASE stands for:**  
a) Computer-Aided Software Engineering  
b) Control Aided Science and Engineering  
c) Cost Aided System Experiments  
d) Computer Automated System Engineering  
**Answer: a) Computer-Aided Software Engineering**  

**Q32. Upper CASE tools are used in:**  
a) Implementation phase  
b) Early phases (requirements, design)  
c) Testing phase  
d) Maintenance phase  
**Answer: b) Early phases (requirements, design)**  

**Q33. Lower CASE tools are used in:**  
a) Requirements phase  
b) Design phase  
c) Later phases (coding, testing, maintenance)  
d) Planning phase  
**Answer: c) Later phases (coding, testing, maintenance)**  

**Q34. Integrated CASE (I-CASE) tools cover:**  
a) Only requirements  
b) Only coding  
c) Entire SDLC  
d) Only testing  
**Answer: c) Entire SDLC**  

**Q35. Which is a benefit of CASE tools?**  
a) Increased errors  
b) Improved productivity and quality  
c) Slower development  
d) Higher costs  
**Answer: b) Improved productivity and quality**  

**Q36. Which is NOT a CASE tool component?**  
a) Central Repository  
b) Code Generator  
c) Cooking Assistant  
d) Diagram Editor  
**Answer: c) Cooking Assistant**  

**Q37. A central repository in CASE tools stores:**  
a) Only code  
b) All project information and artifacts  
c) Only test cases  
d) Only documents  
**Answer: b) All project information and artifacts**  

**Q38. Code generators in CASE tools:**  
a) Write code automatically from designs  
b) Delete code  
c) Test code  
d) Debug code  
**Answer: a) Write code automatically from designs**  

**Q39. Which is a limitation of CASE tools?**  
a) Improve quality  
b) Increase productivity  
c) High cost and learning curve  
d) Better documentation  
**Answer: c) High cost and learning curve**  

**Q40. Examples of CASE tools include:**  
a) Microsoft Word  
b) Rational Rose, Enterprise Architect  
c) Photoshop  
d) Excel  
**Answer: b) Rational Rose, Enterprise Architect**

**Q63. Cost is which type of measure?**

**Q67. Maintainability Index measures:**
