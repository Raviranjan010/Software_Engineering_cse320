# Software Engineering MCQs - Module 3: Object-Oriented Development and UML

## Part 1: Unified Process

**Q1. The Unified Process has how many phases?**  
a) 3  
b) 4  
c) 5  
d) 6  
**Answer: b) 4**  
*Explanation: Inception, Elaboration, Construction, Transition*

**Q2. Which phase of Unified Process establishes the business case?**  
a) Elaboration  
b) Construction  
c) Inception  
d) Transition  
**Answer: c) Inception**  

**Q3. The Elaboration phase focuses on:**  
a) Deploying the system  
b) Analyzing requirements and creating architecture  
c) Writing all code  
d) Testing only  
**Answer: b) Analyzing requirements and creating architecture**  

**Q4. In which phase is the majority of coding done?**  
a) Inception  
b) Elaboration  
c) Construction  
d) Transition  
**Answer: c) Construction**  

**Q5. The Transition phase involves:**  
a) Requirements gathering  
b) System deployment and user training  
c) Architecture design  
d) Initial planning  
**Answer: b) System deployment and user training**  

**Q6. Unified Process is:**  
a) Linear  
b) Iterative and incremental  
c) Waterfall-based  
d) Agile only  
**Answer: b) Iterative and incremental**  

**Q7. Which workflow in UP includes requirements gathering?**  
a) Design workflow  
b) Requirements workflow  
c) Implementation workflow  
d) Testing workflow  
**Answer: b) Requirements workflow**

## Part 2: Use Case Diagrams

**Q8. Use case diagrams show:**  
a) Class relationships  
b) System functionality from user perspective  
c) Code structure  
d) Database schema  
**Answer: b) System functionality from user perspective**  

**Q9. In use case diagrams, an actor is represented by:**  
a) Rectangle  
b) Stick figure  
c) Ellipse  
d) Diamond  
**Answer: b) Stick figure**  

**Q10. A use case is represented by:**  
a) Rectangle  
b) Stick figure  
c) Ellipse/Oval  
d) Triangle  
**Answer: c) Ellipse/Oval**  

**Q11. An actor can be:**  
a) Only a person  
b) Only a system  
c) A person, system, or device  
d) Only a database  
**Answer: c) A person, system, or device**  

**Q12. The <<include>> relationship means:**  
a) Optional behavior  
b) Mandatory inclusion of another use case  
c) Extension of behavior  
d) Generalization  
**Answer: b) Mandatory inclusion of another use case**  

**Q13. The <<extend>> relationship represents:**  
a) Mandatory behavior  
b) Optional or conditional behavior  
c) Inheritance  
d) Association  
**Answer: b) Optional or conditional behavior**  

**Q14. Generalization in use cases is similar to:**  
a) Aggregation  
b) Inheritance  
c) Composition  
d) Dependency  
**Answer: b) Inheritance**  

**Q15. A use case diagram is part of:**  
a) Dynamic modeling  
b) Behavioral modeling  
c) Structural modeling  
d) Data modeling  
**Answer: b) Behavioral modeling**  

**Q16. The system boundary in use case diagram is shown as:**  
a) Circle  
b) Rectangle  
c) Diamond  
d) Triangle  
**Answer: b) Rectangle**  

**Q17. Multiple actors can be associated with:**  
a) Only one use case  
b) Multiple use cases  
c) No use cases  
d) Only the system  
**Answer: b) Multiple use cases**  

**Q18. Which relationship is shown with a dashed arrow?**  
a) Association  
b) Include and Extend  
c) Generalization  
d) All of the above  
**Answer: b) Include and Extend**  

**Q19. An actor that inherits from another actor is called:**  
a) Primary actor  
b) Secondary actor  
c) General/Special actor  
d) System actor  
**Answer: c) General/Special actor**  

**Q20. Use case diagrams are useful for:**  
a) Code optimization  
b) Capturing functional requirements  
c) Database design  
d) Performance testing  
**Answer: b) Capturing functional requirements**

## Part 3: Class Diagrams

**Q21. Class diagrams represent:**  
a) Dynamic behavior  
b) Static structure  
c) Process flow  
d) Data flow  
**Answer: b) Static structure**  

**Q22. In class diagrams, the '-' symbol represents:**  
a) Public visibility  
b) Private visibility  
c) Protected visibility  
d) Package visibility  
**Answer: b) Private visibility**  

**Q23. The '+' symbol in class diagrams represents:**  
a) Public visibility  
b) Private visibility  
c) Protected visibility  
d) Static method  
**Answer: a) Public visibility**  

**Q24. The '#' symbol represents:**  
a) Public  
b) Private  
c) Protected  
d) Package  
**Answer: c) Protected**  

**Q25. An association relationship is shown as:**  
a) Dashed line  
b) Solid line  
c) Dashed arrow  
d) Diamond  
**Answer: b) Solid line**  

**Q26. Aggregation is represented by:**  
a) Filled diamond  
b) Empty diamond  
c) Solid arrow  
d) Dashed line  
**Answer: b) Empty diamond**  

**Q27. Composition is represented by:**  
a) Filled diamond  
b) Empty diamond  
c) Solid arrow  
d) Dashed arrow  
**Answer: a) Filled diamond**  

**Q28. Inheritance is shown with:**  
a) Solid line with empty triangle  
b) Dashed line with arrow  
c) Solid line with diamond  
d) Dashed line with diamond  
**Answer: a) Solid line with empty triangle**  

**Q29. Dependency relationship is shown as:**  
a) Solid line  
b) Dashed line with open arrow  
c) Solid line with filled diamond  
d) Solid line with empty triangle  
**Answer: b) Dashed line with open arrow**  

**Q30. In aggregation, the part can exist:**  
a) Only with the whole  
b) Independently of the whole  
c) Never exists  
d) Only temporarily  
**Answer: b) Independently of the whole**  

**Q31. In composition, if the whole is destroyed, the part:**  
a) Continues to exist  
b) Is also destroyed  
c) Becomes independent  
d) Is transferred to another whole  
**Answer: b) Is also destroyed**  

**Q32. "Car has an Engine" where engine cannot exist without car is:**  
a) Aggregation  
b) Composition  
c) Association  
d) Dependency  
**Answer: b) Composition**  

**Q33. "Department has Professors" where professors can exist independently is:**  
a) Composition  
b) Aggregation  
c) Inheritance  
d) Dependency  
**Answer: b) Aggregation**  

**Q34. Multiplicity 1..* means:**  
a) Exactly one  
b) Zero or more  
c) One or more  
d) Zero or one  
**Answer: c) One or more**  

**Q35. Multiplicity 0..1 means:**  
a) Exactly one  
b) Zero or one  
c) One or more  
d) Many  
**Answer: b) Zero or one**  

**Q36. Multiplicity * means:**  
a) Zero or more  
b) Exactly one  
c) One or more  
d) At most one  
**Answer: a) Zero or more**  

**Q37. A class diagram compartment contains:**  
a) Only class name  
b) Class name, attributes, and methods  
c) Only methods  
d) Only attributes  
**Answer: b) Class name, attributes, and methods**  

**Q38. Realization relationship is between:**  
a) Two classes  
b) Interface and implementing class  
c) Two objects  
d) Two packages  
**Answer: b) Interface and implementing class**  

**Q39. In "Student enrolls in Course", the relationship is:**  
a) Composition  
b) Aggregation  
c) Association  
d) Dependency  
**Answer: c) Association**  

**Q40. Which relationship represents "is-a" relationship?**  
a) Association  
b) Aggregation  
c) Inheritance  
d) Dependency  
**Answer: c) Inheritance**  

**Q41. Which relationship represents "has-a" relationship?**  
a) Inheritance  
b) Aggregation  
c) Dependency  
d) Realization  
**Answer: b) Aggregation**  

**Q42. A static member in UML is shown as:**  
a) Italic  
b) Underlined  
c) Bold  
d) Strikethrough  
**Answer: b) Underlined**  

**Q43. Abstract class name is shown as:**  
a) Bold  
b) Italic  
c) Underlined  
d) Regular  
**Answer: b) Italic**  

**Q44. <<interface>> stereotype indicates:**  
a) Abstract class  
b) Interface  
c) Concrete class  
d) Enumeration  
**Answer: b) Interface**  

**Q45. In a class diagram, methods are in the:**  
a) Top compartment  
b) Middle compartment  
c) Bottom compartment

## Part 4: Sequence Diagrams

**Q46. Sequence diagrams show:**  
a) Static structure  
b) Object interactions over time  
c) Class relationships  
d) System architecture  
**Answer: b) Object interactions over time**  

**Q47. Lifelines in sequence diagrams are represented by:**  
a) Horizontal lines  
b) Vertical dashed lines  
c) Rectangles  
d) Ellipses  
**Answer: b) Vertical dashed lines**  

**Q48. An activation bar represents:**  
a) Object creation  
b) Time during which an object is performing an operation  
c) Message passing  
d) Object destruction  
**Answer: b) Time during which an object is performing an operation**  

**Q49. Messages in sequence diagrams flow:**  
a) Bottom to top  
b) Top to bottom  
c) Left to right only  
d) Right to left only  
**Answer: b) Top to bottom**  

**Q50. A synchronous message is shown with:**  
a) Open arrowhead  
b) Filled arrowhead  
c) Dashed line  
d) No arrowhead  
**Answer: b) Filled arrowhead**  

**Q51. An asynchronous message is shown with:**  
a) Filled arrowhead  
b) Open arrowhead  
c) Diamond  
d) Circle  
**Answer: b) Open arrowhead**  

**Q52. A return message is shown with:**  
a) Solid line with filled arrow  
b) Dashed line with open arrow  
c) Solid line with open arrow  
d) Dashed line with filled arrow  
**Answer: b) Dashed line with open arrow**  

**Q53. Object creation is shown with:**  
a) <<create>> message  
b) <<destroy>> message  
c) <<new>> message  
d) <<start>> message  
**Answer: a) <<create>> message**  

**Q54. The x symbol on a lifeline represents:**  
a) Object creation  
b) Object destruction  
c) Message send  
d) Message receive  
**Answer: b) Object destruction**  

**Q55. Self-message occurs when:**  
a) Object sends message to another object  
b) Object sends message to itself  
c) System sends message to object  
d) Actor sends message to object  
**Answer: b) Object sends message to itself**

## Part 5: Activity Diagrams

**Q56. Activity diagrams are similar to:**  
a) Class diagrams  
b) Flowcharts  
c) Use case diagrams  
d) Sequence diagrams  
**Answer: b) Flowcharts**  

**Q57. The initial node in activity diagrams is shown as:**  
a) Solid circle  
b) Circle with border  
c) Diamond  
d) Rectangle  
**Answer: a) Solid circle**  

**Q58. The final node is shown as:**  
a) Solid circle  
b) Circle with border (bull's eye)  
c) Diamond  
d) Rectangle with rounded corners  
**Answer: b) Circle with border (bull's eye)**  

**Q59. A decision node is represented by:**  
a) Circle  
b) Rectangle  
c) Diamond  
d) Triangle  
**Answer: c) Diamond**  

**Q60. Fork and join nodes are shown as:**  
a) Thin horizontal or vertical bars  
b) Diamonds  
c) Circles  
d) Rectangles  
**Answer: a) Thin horizontal or vertical bars**  

**Q61. Swimlanes in activity diagrams:**  
a) Show parallel processing  
b) Organize activities by responsible party  
c) Show decisions  
d) Show object flow  
**Answer: b) Organize activities by responsible party**  

**Q62. Activity diagrams are useful for modeling:**  
a) Class structure  
b) Business workflows  
c) Database schema  
d) Network architecture  
**Answer: b) Business workflows**  

**Q63. A merge node:**  
a) Combines concurrent flows  
b) Combines alternative flows  
c) Splits flow into concurrent paths  
d) Ends the activity  
**Answer: b) Combines alternative flows**  

**Q64. The difference between fork and merge is:**  
a) Fork splits concurrent, merge combines alternative  
b) Fork splits alternative, merge combines concurrent  
c) Both are same  
d) Neither splits nor combines  
**Answer: a) Fork splits concurrent, merge combines alternative**  

**Q65. Object nodes in activity diagrams show:**  
a) Control flow  
b) Data flow between activities  
c) Decisions  
d) Parallel processing  
**Answer: b) Data flow between activities**

**Q66. Model validation ensures:**  
a) Code is bug-free  
b) Models are correct, complete, and consistent  
c) System is deployed  
d) Requirements are gathered  
**Answer: b) Models are correct, complete, and consistent**  

**Q67. Traceability in model validation means:**  
a) Models can be traced to requirements  
b) Models have colors  
c) Models are large  
d) Models are complex  
**Answer: a) Models can be traced to requirements**  

**Q68. Consistency checking verifies:**  
a) No contradictions between different diagrams  
b) Models are beautiful  
c) Models are complete  
d) Models are fast  
**Answer: a) No contradictions between different diagrams**  

**Q69. Completeness checking ensures:**  
a) All required elements are present  
b) Models are consistent  
c) Models are traceable  
d) Models are simple  
**Answer: a) All required elements are present**  

**Q70. Coding standards help in:**  
a) Making code colorful  
b) Maintaining consistency and readability  
c) Increasing complexity  
d) Reducing performance  
**Answer: b) Maintaining consistency and readability**  

**Q71. Model reviews are conducted to:**  
a) Find defects in models early  
b) Write code  
c) Deploy system  
d) Gather requirements  
**Answer: a) Find defects in models early**  

**Q72. A use case not implemented in code indicates:**

**Q73. Which tool helps in maintaining traceability?**

**Q74. Syntax checking in UML verifies:**

**Q75. Semantic checking verifies:**

## Part 7: Advanced UML Concepts

**Q76. Which UML diagram is NOT behavioral?**  
a) Use Case Diagram  
b) Sequence Diagram  
c) Class Diagram  
d) Activity Diagram  
**Answer: c) Class Diagram**  
*Explanation: Class diagram is structural, not behavioral.*

**Q77. Which UML diagram is NOT structural?**  
a) Class Diagram  
b) Component Diagram  
c) Sequence Diagram  
d) Deployment Diagram  
**Answer: c) Sequence Diagram**  
*Explanation: Sequence diagram is behavioral.*

**Q78. The most important UML diagram in software engineering is:**  
a) Use Case Diagram  
b) Class Diagram  
c) Activity Diagram  
d) State Diagram  
**Answer: b) Class Diagram**  

**Q79. Package diagrams are used to:**  
a) Show system organization into packages  
b) Show object interactions  
c) Show workflows  
d) Show class attributes  
**Answer: a) Show system organization into packages**  

**Q80. Component diagrams show:**  
a) Physical components and their relationships  
b) Class relationships  
c) Object interactions  
d) Workflows  
**Answer: a) Physical components and their relationships**  

**Q81. Deployment diagrams show:**  
a) Hardware nodes and software artifacts  
b) Class hierarchies  
c) Use case relationships  
d) Activity flows  
**Answer: a) Hardware nodes and software artifacts**  

**Q82. State machine diagrams model:**  
a) Object states and transitions  
b) Class relationships  
c) System deployment  
d) Package organization  
**Answer: a) Object states and transitions**  

**Q83. Communication diagrams are similar to sequence diagrams but emphasize:**  
a) Time ordering  
b) Object relationships and message flow  
c) Class structure  
d) Workflows  
**Answer: b) Object relationships and message flow**  

**Q84. Timing diagrams focus on:**  
a) Exact timing constraints  
b) Class relationships  
c) Use cases  
d) Activities  
**Answer: a) Exact timing constraints**  

**Q85. Interaction overview diagrams combine:**  
a) Activity and sequence diagrams  
b) Class and use case diagrams  
c) State and deployment diagrams  
d) Component and package diagrams  
**Answer: a) Activity and sequence diagrams**  

**Q86. Object diagrams show:**  
a) Instances of classes at a particular moment  
b) Class structure  
c) System deployment  
d) Workflows  
**Answer: a) Instances of classes at a particular moment**  

**Q87. Composite structure diagrams show:**  
a) Internal structure of a class  
b) System deployment  
c) Use case relationships  
d) Activity flows  
**Answer: a) Internal structure of a class**  

**Q88. UML stands for:**  
a) Unified Modeling Language  
b) Universal Modeling Language  
c) Unified Meta Language  
d) Universal Meta Language  
**Answer: a) Unified Modeling Language**  

**Q89. UML was standardized by:**  
a) IEEE  
b) ISO  
c) OMG (Object Management Group)

**Answer: c) OMG (Object Management Group)**  

**Q90. How many UML diagram types are there in UML 2.x?**
