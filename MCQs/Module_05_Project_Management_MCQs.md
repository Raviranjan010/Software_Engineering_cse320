# Software Engineering MCQs - Module 5: Project Management and DevOps

## Part 1: Project Management

**Q1. Software Project Management involves:**  
a) Only coding  
b) Planning, monitoring, and controlling software projects  
c) Only testing  
d) Only deployment  
**Answer: b) Planning, monitoring, and controlling software projects**  

**Q2. The triple constraint in project management is:**  
a) Time, Cost, Quality  
b) Scope, Time, Cost  
c) Quality, Risk, Resources  
d) Time, Risk, Scope  
**Answer: b) Scope, Time, Cost**  

**Q3. A Work Breakdown Structure (WBS) is:**  
a) List of team members  
b) Hierarchical decomposition of project work  
c) Budget document  
d) Risk register  
**Answer: b) Hierarchical decomposition of project work**  

**Q4. Risk Management includes:**  
a) Only avoiding risks  
b) Identification, analysis, planning, and monitoring  
c) Ignoring risks  
d) Transferring all risks  
**Answer: b) Identification, analysis, planning, and monitoring**  

**Q5. Which is NOT a risk response strategy?**  
a) Avoidance  
b) Mitigation  
c) Ignorance  
d) Transfer  
**Answer: c) Ignorance**  

**Q6. The critical path in a project is:**  
a) Shortest path  
b) Longest path determining minimum project duration  
c) Path with most resources  
d) Path with least cost  
**Answer: b) Longest path determining minimum project duration**  

**Q7. Slack/Float time is:**  
a) Time an activity can be delayed without delaying project  
b) Time spent on breaks  
c) Overtime work  
d) Idle time  
**Answer: a) Time an activity can be delayed without delaying project**  

**Q8. Gantt Charts show:**  
a) Project schedule with activities over time  
b) Code structure  
c) Network diagram  
d) Risk matrix  
**Answer: a) Project schedule with activities over time**  

**Q9. PERT stands for:**  
a) Program Evaluation and Review Technique  
b) Project Estimation and Review Tool  
c) Program Execution and Review Technique  
d) Project Evaluation and Reporting Tool  
**Answer: a) Program Evaluation and Review Technique**  

**Q10. CPM stands for:**  
a) Critical Path Method  
b) Cost Planning Method  
c) Critical Planning Model  
d) Cost Path Method  
**Answer: a) Critical Path Method**  

**Q11. In PERT, expected time formula is:**  
a) (Optimistic + Pessimistic + Most Likely) / 3  
b) (Optimistic + 4×Most Likely + Pessimistic) / 6  
c) (Optimistic + Pessimistic) / 2  
d) (Most Likely + Pessimistic) / 2  
**Answer: b) (Optimistic + 4×Most Likely + Pessimistic) / 6**  

**Q12. Which technique uses probabilistic time estimates?**  
a) CPM  
b) PERT  
c) Gantt Chart  
d) WBS  
**Answer: b) PERT**  

**Q13. Which technique uses deterministic time estimates?**  
a) PERT  
b) CPM  
c) Monte Carlo  
d) Delphi  
**Answer: b) CPM**  

**Q14. Resource leveling is done to:**  
a) Increase resources  
b) Balance resource usage over time  
c) Reduce project scope  
d) Increase project cost  
**Answer: b) Balance resource usage over time**  

**Q15. Project crashing involves:**  
a) Adding resources to shorten duration  
b) Reducing scope  
c) Increasing risks  
d) Delaying project  
**Answer: a) Adding resources to shorten duration**

## Part 2: Cost Estimation

**Q16. Cost estimation predicts:**  
a) Only time  
b) Effort, cost, and schedule  
c) Only quality  
d) Only risks  
**Answer: b) Effort, cost, and schedule**  

**Q17. LOC stands for:**  
a) Lines of Code  
b) Level of Complexity  
c) Loss of Control  
d) List of Components  
**Answer: a) Lines of Code**  

**Q18. KLOC means:**  
a) Kilo Lines of Code (Thousand lines)  
b) Key Lines of Code  
c) Known Lines of Code  
d) Kernel Lines of Code  
**Answer: a) Kilo Lines of Code (Thousand lines)**  

**Q19. Function Point Analysis measures:**  
a) Code size  
b) Functionality from user's perspective  
c) Execution time  
d) Memory usage  
**Answer: b) Functionality from user's perspective**  

**Q20. FPA is independent of:**  
a) Requirements  
b) Programming language  
c) Users  
d) Features  
**Answer: b) Programming language**  

**Q21. Which is NOT a component of Function Point Analysis?**  
a) External Inputs  
b) External Outputs  
c) Internal Logical Files  
d) Lines of Code  
**Answer: d) Lines of Code**  

**Q22. The COCOMO model was developed by:**  
a) Bill Gates  
b) Barry Boehm  
c) Margaret Hamilton  
d) Ken Schwaber  
**Answer: b) Barry Boehm**  

**Q23. COCOMO stands for:**  
a) Cost Constructive Estimation Model  
b) Constructive Cost Estimating Model  
c) Constructive Cost Estimation Model  
d) Computer Cost Model  
**Answer: b) Constructive Cost Estimating Model**  

**Q24. How many modes does Basic COCOMO have?**  
a) 2  
b) 3  
c) 4  
d) 5  
**Answer: b) 3**  
*Explanation: Organic, Semi-detached, Embedded*

**Q25. Organic mode in COCOMO is for:**  
a) Large, complex projects  
b) Small, simple projects with experienced teams  
c) Real-time systems  
d) Military projects  
**Answer: b) Small, simple projects with experienced teams**  

**Q26. Embedded mode in COCOMO is for:**  
a) Simple projects  
b) Complex projects with tight constraints  
c) Medium projects  
d) Web applications  
**Answer: b) Complex projects with tight constraints**  

**Q27. Semi-detached mode is for:**  
a) Very simple projects  
b) Medium complexity projects with mixed teams  
c) Very complex projects  
d) Trivial projects  
**Answer: b) Medium complexity projects with mixed teams**  

**Q28. In Basic COCOMO, Effort (E) formula is:**  
a) E = a × (KLOC)^b  
b) E = a + b × KLOC  
c) E = a × KLOC / b  
d) E = a^b × KLOC  
**Answer: a) E = a × (KLOC)^b**  

**Q29. In Basic COCOMO, Time (T) formula is:**  
a) T = c × (E)^d  
b) T = c + d × E  
c) T = c × E / d  
d) T = c^d × E  
**Answer: a) T = c × (E)^d**  

**Q30. For Organic mode, COCOMO coefficients are:**  
a) a=2.4, b=1.05, c=2.5, d=0.38  
b) a=3.0, b=1.12, c=2.5, d=0.35  
c) a=3.6, b=1.20, c=2.5, d=0.32  
d) a=2.0, b=1.0, c=2.0, d=0.3  
**Answer: a) a=2.4, b=1.05, c=2.5, d=0.38**  

**Q31. For Embedded mode, COCOMO coefficients are:**  
a) a=2.4, b=1.05  
b) a=3.0, b=1.12  
c) a=3.6, b=1.20  
d) a=4.0, b=1.30  
**Answer: c) a=3.6, b=1.20**  

**Q32. If Effort = 100 PM and Time = 10 months, Persons Required = ?**  
a) 10  
b) 100  
c) 1000  
d) 1  
**Answer: a) 10**  
*Explanation: Persons = E / T = 100 / 10 = 10*

**Q33. FP formula includes value adjustment factors. How many factors?**  
a) 10  
b) 12  
c) 14  
d) 16  
**Answer: c) 14**  

**Q34. Function Point formula is:**  
a) FP = Count Total × [0.65 + 0.01 × SUM(Fi)]  
b) FP = Count Total + SUM(Fi)  
c) FP = Count Total / SUM(Fi)  
d) FP = Count Total - SUM(Fi)  
**Answer: a) FP = Count Total × [0.65 + 0.01 × SUM(Fi)]**  

**Q35. Which estimation technique is MOST accurate?**  
a) LOC  
b) Expert judgment  
c) Function Point Analysis  
d) COCOMO II (detailed)  
**Answer: d) COCOMO II (detailed)**

## Part 4: Scheduling Techniques

**Q46. The Critical Path Method (CPM) helps to:**  
a) Find shortest path  
b) Identify activities that cannot be delayed  
c) Reduce costs  
d) Increase resources  
**Answer: b) Identify activities that cannot be delayed**  

**Q47. Activities on critical path have:**  
a) Maximum slack  
b) Zero slack  
c) Negative slack  
d) Variable slack  
**Answer: b) Zero slack**  

**Q48. If an activity on critical path is delayed, the project:**  
a) Finishes early  
b) Is delayed  
c) Not affected  
d) Costs less  
**Answer: b) Is delayed**  

**Q49. A network diagram shows:**  
a) Activities and their dependencies  
b) Code structure  
c) Team hierarchy  
d) Budget breakdown  
**Answer: a) Activities and their dependencies**  

**Q50. Early Start (ES) is:**  
a) Earliest time an activity can start  
b) Latest time an activity can start  
c) Actual start time  
d) Planned start time  
**Answer: a) Earliest time an activity can start**  

**Q51. Late Finish (LF) is:**  
a) Earliest time an activity can finish  
b) Latest time an activity can finish without delaying project  
c) Actual finish time  
d) Planned finish time  
**Answer: b) Latest time an activity can finish without delaying project**  

**Q52. Forward Pass calculates:**  
a) LF and LS  
b) ES and EF  
c) Slack  
d) Critical path  
**Answer: b) ES and EF**  

**Q53. Backward Pass calculates:**  
a) ES and EF  
b) LF and LS  
c) Duration  
d) Dependencies  
**Answer: b) LF and LS**  

**Q54. Slack = LF - EF or LS - ES. If Slack = 0, activity is:**  
a) Non-critical  
b) Critical  
c) Optional  
d) Complete  
**Answer: b) Critical**  

**Q55. In PERT, variance is calculated as:**  
a) [(Pessimistic - Optimistic) / 6]^2  
b) (Pessimistic - Optimistic) / 6  
c) (Pessimistic + Optimistic) / 2  
d) Most Likely / 6  
**Answer: a) [(Pessimistic - Optimistic) / 6]^2**  

**Q56. Gantt charts are useful for:**  
a) Showing project schedule visually  
b) Calculating critical path  
c) Risk analysis  
d) Cost estimation  
**Answer: a) Showing project schedule visually**  

**Q57. Resource histogram shows:**  
a) Resource allocation over time  
b) Project costs  
c) Risks  
d) Dependencies  
**Answer: a) Resource allocation over time**  

**Q58. Fast tracking means:**  
a) Doing activities in parallel that were sequential  
b) Adding more resources  
c) Reducing scope  
d) Canceling activities  
**Answer: a) Doing activities in parallel that were sequential**  

**Q59. Crashing means:**  
a) Doing activities in parallel  
b) Adding resources to critical path activities  
c) Reducing quality  
d) Extending deadline  
**Answer: b) Adding resources to critical path activities**  

**Q60. Milestone is:**  
a) An activity with duration  
b) Significant point or event with zero duration  
c) A resource  
d) A risk  
**Answer: b) Significant point or event with zero duration**

## Part 5: Team Organization and Agile

**Q61. Democratic Decentralized (DD) team:**  
a) Has permanent leader  
b) Has no permanent leader  
c) Has strict hierarchy  
d) Is controlled  
**Answer: b) Has no permanent leader**  

**Q62. Controlled Centralized (CC) team:**  
a) Has no leader  
b) Has defined leader who coordinates all tasks  
c) Has multiple leaders  
d) Is democratic  
**Answer: b) Has defined leader who coordinates all tasks**  

**Q63. Controlled Decentralized (CD) team:**  
a) Has no leader  
b) Has defined leader with secondary leaders for subtasks  
c) Has permanent leader only  
d) Is fully centralized  
**Answer: b) Has defined leader with secondary leaders for subtasks**  

**Q64. Agile Manifesto values:**

## Part 6: Advanced Project Management

**Q76. Earned Value Management (EVM) integrates:**  
a) Scope, schedule, and cost  
b) Quality and risk  
c) Resources and time  
d) Scope and quality  
**Answer: a) Scope, schedule, and cost**  

**Q77. SPI (Schedule Performance Index) > 1 means:**  
a) Behind schedule  
b) Ahead of schedule  
c) On schedule  
d) No schedule  
**Answer: b) Ahead of schedule**  

**Q78. CPI (Cost Performance Index) < 1 means:**  
a) Under budget  
b) Over budget  
c) On budget  
d) No budget  
**Answer: b) Over budget**  

**Q79. A risk register contains:**  
a) Team members  
b) Identified risks with analysis and responses  
c) Code repository  
d) Test cases  
**Answer: b) Identified risks with analysis and responses**  

**Q80. Monte Carlo simulation is used for:**  
a) Code review  
b) Risk analysis with probabilistic modeling  
c) Testing  
d) Deployment  
**Answer: b) Risk analysis with probabilistic modeling**  

**Q81. The Delphi technique is used for:**  
a) Coding

**Q82. Brooks' Law states:**

**Q85. Stakeholder analysis identifies:**

**Q86. Quality Assurance (QA) focuses on:**

**Q87. Quality Control (QC) focuses on:**

**Answer: b) Identifying defects in deliverables**  

**Q88. Configuration Audit verifies:**

**Q89. Change Control Board (CCB) is responsible for:**

**Q90. Release Planning in Agile determines:**

## Part 7: DevOps and CI/CD

**Q91. DevOps combines:**  
a) Development and Operations  
b) Design and Testing  
c) Planning and Monitoring  
d) Coding and Documentation  
**Answer: a) Development and Operations**  

**Q92. Infrastructure as Code (IaC) means:**  
a) Writing infrastructure manually  
b) Managing infrastructure through machine-readable files  
c) Using physical servers only  
d) No automation  
**Answer: b) Managing infrastructure through machine-readable files**  

**Q93. Continuous Integration (CI) means:**  
a) Integrating code rarely  
b) Frequently merging code changes to main branch  
c) Never integrating  
d) Integrating at end only  
**Answer: b) Frequently merging code changes to main branch**  

**Q94. Continuous Deployment (CD) means:**  
a) Manual deployment  
b) Automated deployment to production  
c) No deployment  
d) Delayed deployment  
**Answer: b) Automated deployment to production**  

**Q95. A build pipeline includes:**  
a) Only compilation  
b) Compile, test, package, deploy stages  
c) Only testing  
d) Only deployment  
**Answer: b) Compile, test, package, deploy stages**  

**Q96. Jenkins is a:**  
a) Version control tool  
b) CI/CD automation server  
c) Code editor  
d) Database  
**Answer: b) CI/CD automation server**  

**Q97. Docker is used for:**  
a) Version control  
b) Containerization  
c) Testing  
d) Documentation  
**Answer: b) Containerization**  

**Q98. Kubernetes is used for:**  
a) Version control  
b) Container orchestration  
c) Code editing  
d) Database management  
**Answer: b) Container orchestration**  

**Q99. Blue-Green Deployment:**  
a) Uses single environment  
b) Uses two identical environments for zero-downtime deployment

**Answer: b) Uses two identical environments for zero-downtime deployment**
