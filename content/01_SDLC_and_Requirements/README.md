# Module 1: SDLC and Requirements Engineering

## Overview
This module covers the foundational concepts of Software Engineering, various SDLC models, and requirements engineering processes. This is the cornerstone of the course and carries significant weight in examinations.

## Topics Covered

### 1. Introduction to Software Engineering
- Evolution and history
- Software crisis and its impact
- Software characteristics
- Software Engineering vs Programming
- **File**: [01_Introduction_to_SE.md](01_Introduction_to_SE.md)

### 2. SDLC Models
- Waterfall Model
- Prototyping Model
- Spiral Model
- V-Model
- Agile & Scrum
- DevOps Lifecycle
- **File**: [02_SDLC_Models.md](02_SDLC_Models.md)

### 3. Requirements Engineering
- Functional vs Non-functional requirements
- Requirements gathering techniques
- Requirements analysis
- Elicitation methods
- **File**: [03_Requirements_Engineering.md](03_Requirements_Engineering.md)

### 4. Software Requirements Specification (SRS)
- IEEE 830 Standard
- Writing effective requirements
- Validation techniques
- SRS template and examples
- **File**: [04_SRS_Document.md](04_SRS_Document.md)

## Course Outcome: CO1
After completing this module, you will be able to:
- Explain the evolution of Software Engineering
- Apply appropriate SDLC models to requirement-based problems
- Analyze and document software requirements effectively

## Exam Weightage: ~20-25%
