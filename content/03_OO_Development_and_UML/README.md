# Module 3: Object-Oriented Development and UML

## Overview
This module covers object-oriented software development using Unified Process and UML (Unified Modeling Language) diagrams - essential for system design and analysis.

## Topics Covered

### 1. Unified Process
- Phases: Inception, Elaboration, Construction, Transition
- Core workflows
- Iterative and incremental development
- **File**: [01_Unified_Process.md](01_Unified_Process.md)

### 2. Use Case Diagrams ⭐
- Actors, use cases, relationships
- Include, extend, generalize
- **File**: [02_UML_UseCase_Diagrams.md](02_UML_UseCase_Diagrams.md)

### 3. Class Diagrams ⭐⭐ **VERY IMPORTANT**
- Classes, attributes, methods
- Relationships (association, aggregation, composition, inheritance)
- Multiplicity
- **File**: [03_UML_Class_Diagrams.md](03_UML_Class_Diagrams.md)

### 4. Sequence Diagrams ⭐
- Lifelines, messages, activation bars
- Time ordering
- **File**: [04_UML_Sequence_Diagrams.md](04_UML_Sequence_Diagrams.md)

### 5. Activity Diagrams
- Actions, decisions, forks, joins
- Swimlanes
- **File**: [05_UML_Activity_Diagrams.md](05_UML_Activity_Diagrams.md)

### 6. Model Validation
- Consistency and completeness checking
- Traceability
- Coding standards
- **File**: [06_Model_Validation.md](06_Model_Validation.md)

## Course Outcome: CO3
After completing this module, you will be able to:
- Use UML models to represent systems
- Analyze model correctness and consistency
- Create various UML diagrams for system design

## Exam Weightage: ~20-25%
